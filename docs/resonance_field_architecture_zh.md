## 1. 设计哲学

SylannEngine 不是神经网络，不是规则引擎，不是情感词典。它是一个**代数动力系统**：

- 7 个计算模块构成完全 6-单纯形 Δ⁶ 的顶点
- 模块之间通过 441 条有向耦合通道相互影响
- 系统通过迭代共振自发收敛到稳态（不是顺序计算）
- 表达不是"算出来的"，而是系统逃离吸引子时的**相变事件**

类比：不是"计算情绪"，而是"让情绪自己涌现"。

## 4. 核心动力学机制

### 4.1 Hebbian 可塑性（用进废退）

```
w(t+1) = w(t) + η·activation·trace - λ·w(t)
```

- **η（学习率）**：由 openness 人格维度控制
- **λ（衰减率）**：由 conscientiousness 控制
- **稳态缩放**：总权重预算守恒（Turrigiano 2008）
- **修剪**：权重低于阈值的通道被剪除（Edelman 1987 神经达尔文主义）

**效果**：经常共同激活的模块之间连接变强，不用的连接萎缩。系统自动发现哪些模块组合对当前人格最重要。

### 4.2 高阶 Kuramoto 相位同步

```
dθ_i/dt = ω_i + K₁·Σ w_ij·sin(θ_j - θ_i)           [两体]
                + K₂·Σ sin(θ_j + θ_k - 2θ_i)          [三体, Millán 2020]
                + K₃·Σ sin(Σθ_others - n·θ_i)          [高阶]
```

- **K₁, K₂, K₃**：由 openness 人格维度控制
- **序参量 r**：衡量全局同步程度。r→1 = 模块协调一致 = 可以表达
- **爆炸性同步**：高阶项导致突然的相变（从无序到有序），这就是"灵感涌现"的数学模型

### 4.3 自由能最小化（Friston 2010）

```
F = ½·π·Σ(observed_i - belief_i)²
belief_i(t+1) = belief_i(t) + α·π·(observed_i - belief_i)
```

- **π（精度）**：由 neuroticism 控制。高神经质 = 高精度 = 对预测误差更敏感
- **效果**：系统持续更新内部模型以减少"惊讶"。当惊讶无法被消除时，驱动表达（主动推理）

### 4.4 Hopfield 吸引子景观

```
E = -½·Σ_μ (X·ξ_μ)²
ΔX = γ·Σ_μ (X·ξ_μ)·ξ_μ
```

- **ξ_μ**：存储的吸引子模式（系统反复访问的状态）
- **γ（吸引强度）**：由 extraversion 控制
- **最大吸引子数**：由 patience 控制
- **效果**：系统有"情感记忆"——倾向于回到熟悉的情感模式。表达 = 逃离吸引子。

### 4.5 谐波身份（灵魂）

```
h_id(t) = 0.95·h_id(t-1) + 0.05·Π_{ker L₁}[X(t)]
恢复力: Δx += 0.03·(h_id - x)
```

- **Hodge Laplacian L₁** 的零空间 = 拓扑不变量
- **效果**：无论外部刺激如何扰动，系统总是被轻柔地拉回其"本性"。这就是人格的数学实现——不是参数表，而是拓扑结构。
- **惯性**：由 conscientiousness 控制
- **范数上限**：由 sovereignty_guard 控制

### 4.6 回声状态储备池（时间记忆）

```
r(t) = 0.9·r(t-1) + 0.3·tanh(X(t))    [有外部输入时]
r(t) = 0.9·r(t-1)                        [无外部输入时]
注入: Δx += 0.05·r
```

- **效果**：场"记得"过去的输入。当前状态受历史的衰减影响。
- **关键设计**：只有外部注入才更新储备池，内部 Kuramoto 振荡不算。确保时间记忆反映真实交互历史。

## 5. 表达决策（分岔模型）

表达不是阈值判断，而是 **OR 门分岔**——任何一个强信号都能触发：

| 触发器 | 公式 | 含义 |
|--------|------|------|
| 惊讶 | 1.5 × surprise | 输入出乎预料 |
| 新奇 | attractor_dist × 3 | 场处于未知领域 |
| 点火 | Δ(sync_order) × 5 | 模块突然同步 |
| 原始驱动 | (|M6| - avg) × 2 | 表达模块相对激活 |

```
drive = max(surprise, novelty, ignition, raw) × (0.3 + 0.7·Φ)
```

- **Φ（整合信息）**：意义门控。低 Φ = 噪声，不触发。高 Φ = 有意义的信号。
- **HGT 抑制**：d[3] > 0.75 时 drive × 0.2（自上而下的"别说话"信号）
- **阈值衰减**：沉默越久，阈值越低（压力积累）

## 7. 稳定性保证

### 为什么不会死循环/爆炸

1. **tanh 饱和**：每次迭代后 |x| ≤ 1，硬上限
2. **耗散**：每次迭代 ×0.98，能量持续流失
3. **残余衰减**：每次调用前 ×0.7，上轮 30% 能量蒸发
4. **稳态缩放**：Hebbian 权重总预算守恒，不会无限增长

**实测**：即使每 tick 对全部模块注入最大信号，能量稳定在 23.0（理论上限 28.0）。

### Lyapunov 稳定性定理

$$V(X) = \frac{1}{2}\sum_i \|x_i\|^2$$

在 tanh + 耗散条件下：$\dot{V} \leq -\gamma V + \delta$，所有轨迹进入并停留在紧集 $\{V \leq \delta/\gamma\}$ 内。

## 8. 开发者集成指南

### 8.1 快速开始

```python
from sylanne_core.compute.resonance_integration import ResonanceSpine
from sylanne_core.config import build_profile

# 选择档位
spine = ResonanceSpine(profile=build_profile("lite"))

# 设置人格
spine.apply_personality({
    "extraversion": 0.7,
    "neuroticism": 0.3,
    "openness": 0.8,
    "conscientiousness": 0.5,
    "agreeableness": 0.6,
})

# 处理消息
result = spine.process("我今天很开心", timestamp=time.time())

# 读取结果
result["should_express"]           # bool: 是否应该说话
result["emotion"]                  # dict: 当前情感状态
result["resonance"]["sync_order"]  # float: 场同步度
result["resonance"]["phi"]         # float: 整合信息量

# 反馈
spine.feedback("accepted")  # 或 "ignored", "rejected"

# 持久化
state = spine.to_dict()     # 保存
spine.from_dict(state)      # 恢复
```

### 8.2 平台要求

| 档位 | 最低配置 | 推荐部署 | 延迟 | 并发会话 |
|------|----------|----------|------|----------|
| lite | 任意 CPU, 64MB | 嵌入式 / 实时推理 | ~5ms | 50+ 会话/核 |
| pro | 2核, 256MB, numpy | 桌面 / 云 VM | ~40ms | ~6 会话/核 |
| max | 4核或 GPU, 1GB+ | 研究 / 多智能体 | ~50ms CPU | ~5 会话/核 |

**lite 档：** 默认档位，适合资源受限或高并发场景。原因：
- CPU 往往是共享部署里的稀缺资源
- lite 的 42 通道 + 5ms 延迟已经产生完整的情感动态（Hebbian、Kuramoto、Hopfield 全部工作）
- 并发能力（50+ 会话/核）在多用户场景下比单会话深度更重要
- pro/max 的高阶多体交互是锦上添花，不是必需品

**pro / max 档：** 开发者自行选择，自行承担性能成本。适用于：
- 桌面应用（pro：单用户，深度交互）
- 研究仿真（max：多智能体情感传染实验）
- GPU 服务器（max + torch：批量处理）
